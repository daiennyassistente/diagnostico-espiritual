# Validações do fluxo PIX corrigido

- transaction_control foi garantida no banco com `transactionId`, `resultId`, `status`, `processed` e `emailSent`.
- O webhook do Mercado Pago agora usa `external_reference` como `transactionId`.
- O webhook busca a transação, exige `status = approved`, usa `resultId` para recuperar o diagnóstico correto e evita reprocessamento quando `processed = true`.
- O email com PDF personalizado é enviado apenas no webhook, e `finalizeTransaction(transactionId)` é chamado após sucesso.
- O endpoint `GET /check-payment?transaction_id=...` responde com `{ "status": ... }`.
- O frontend faz polling a cada 3 segundos e redireciona para `/sucesso?transaction_id=...`.
- A rota `/sucesso` foi adicionada como alias de `CheckoutSuccess`.
- Teste automatizado aprovado: `pnpm exec vitest run server/transaction-control.test.ts`.
- Teste automatizado aprovado: `pnpm exec vitest run server/mercadopago-webhook-idempotency.test.ts`.
- Verificação manual aprovada: `curl http://localhost:3000/check-payment?transaction_id=txn-inexistente` retornou `{"status":"pending"}`.
- Verificação manual aprovada: rota `/sucesso?transaction_id=txn-inexistente` renderiza a tela de sucesso sem redirecionar para o quiz.
